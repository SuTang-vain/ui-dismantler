# 三大队 · 人物关系与剧情全解析 — 设计规范

> 源自《三大队》案例页（双基准缩放全视口自适应 / WISE ≤768/480 / 极端 ≤320），用于"影视人物图谱 + 故事脉络 + 作品"垂类的统一组件库建设。本案例采用**故事切换条 + 每段独立放射图谱 + 作品推荐**范式。

### 1.2 语义分组

- **文字层级**：`--sg-ink` > `--sg-muted` > `--sg-subtle`（三级灰阶）
- **品牌色**：`--sg-primary`（蓝紫）为主功能色，`--sg-primary-light` 为脉冲/选中态
- **关系语义五色**：红/粉/金/绿/深灰，连线 dasharray 进一步区分（实线/点线/长短虚线）
- **关系重映射**：`family`→战友（红）、`romance`→家人（粉）、`master`→师徒（金）、`friend`→同行（绿）、`enemy`→对立（深灰）

### 1.3 渐变与叠加

| 用途 | 值 |
|---|---|
| 画布底色 | `linear-gradient(160deg, var(--sg-bg-1) 0%, var(--sg-bg-2) 60%, var(--sg-bg-3) 100%)` |
| 图谱径向光晕 | `radial-gradient(circle at 50% 44%, rgba(100,135,250,.07), transparent 62%)` 叠 `var(--sg-paper)` |
| 故事按钮激活 / 主 Tab | `linear-gradient(135deg, var(--sg-primary), var(--sg-primary-light))` |
| 头像兜底 / 角色标签兜底 | `linear-gradient(135deg, var(--sg-primary), #7b96f7)` + 首字 |
| 作品封面蒙版 | `linear-gradient(180deg, transparent 55%, rgba(0,0,0,0.5) 100%)` |

> 除 `:root` 内令牌定义外，业务样式不得出现硬编码 #hex，统一走 `var(--sg-*)`。

## 二、双基准等比缩放画布

源页核心范式：把整个 `.sg-frame` 当作一张固定尺寸的"画布"，按视口等比缩放居中，保证任意视口下布局比例一致。

| 基准 | 尺寸 | 适用 |
|---|---|---|
| `BASE_LARGE` | 788 × 492 | 宽屏 / 横屏 |
| `BASE_SMALL` | 380 × 456 | 紧凑卡（w<415 或 h<456）/ 窄手机竖屏（w≤768 且 w/h<1） |

**`_shouldUseSmallBase(w,h)`**：紧凑卡 OR 窄手机竖屏 → 选 SMALL，否则 LARGE。

**`_applyScale()`**：
- `scale = min(w/base.w, h/base.h)`
- `.sg-frame` 设为 `base.w × base.h`，`transform: translate(offsetX,offsetY) scale(scale)` 居中
- 切换 canvas 类：`is-small-canvas` / `is-large-canvas` / `is-tiny-canvas`（w≤330 或 h≤380）
- resize 同步触发，内部重排 150ms 防抖

**关键**：所有几何计算（measure / 拖动位移 / 边坐标）都需 `/scaleFactor` 还原到逻辑像素，否则缩放下错位。

## 三、底部 Tab 元素

底部 N 等分 Tab Bar（`flex` 布局）：

```
[ 故事图谱 ] [ 作品推荐 ]
```

- 默认两视图（graph / works），可通过 `options.tabs` 调整
- 激活态：背景填充主色渐变、文字白色、阴影 `0 6px 18px rgba(100,135,250,0.44)`
- 未激活：白底 + 主色描边 `2px rgba(100,135,250,0.35)` + `--sg-primary-dark` 文字

属性约定：`role="tablist"` + `role="tab"` + `aria-selected` + `aria-controls`；面板 `role="tabpanel"` + `aria-labelledby` + `hidden`；切换同步 aria/hidden/active，并触发对应视图渲染（graph 启动物理循环，works 停止）。

## 四、故事图谱视图（graph）

### 4.1 故事切换条

位于图谱上方，横向滚动 + `scroll-snap-type: x proximity`，按钮 `min-width: 122px`，带序号徽标 `.sg-story-index`：

```
[ ① 恶性命案 ] [ ② 漫漫刑途 ] [ ③ 千里追凶 ] [ ④ 终有回响 ]
```

- 激活按钮：主色渐变填充 + 白字 + 强阴影
- 未激活：白底 + 主色描边 + 主色深文字
- 切换条下方 `.sg-story-desc` 卡片显示当前故事描述（半透明白底）
- WISE 右上角出现 `.sg-story-scroll-hint` 滚动提示（`storyHintNudge` 抖动动画），不可右滑时隐藏

### 4.2 放射式关系图谱（核心算法）

切换故事时按 `storyModule.chars` + `storyModule.edges` 重渲染节点与连线。算法链：

1. **`_measure()`**：取 `.sg-graph-wrap` 的逻辑尺寸（除以 scaleFactor），减去面板让位宽（PC 320 / 移动 0）得 `bounds`。
2. **`_graphAvatarMetrics(denseGraph)`**：按 `_currentBase` 尺寸档位返回头像/字号（dense-graph 时更小）。
3. **`_orderRing(others, ringLinks)`**：环形顺序优化。对非主角节点做全排列（n≤8），代价函数 = 边距离惩罚 + 边交叉惩罚，取最小代价顺序，减少连线交叉。
4. **`_computeLayout()`**：主角居中 `(cx,cy)`，其余按 orderRing 顺序在椭圆 `(rx,ry)` 上均匀分布；然后 **120 次迭代避让**（节点 rect 重叠则沿 x/y 推开，含 margin），最后 clamp 到画布边界（含 legendClearY 避让图例）。
5. **`_buildGraph()`**：创建 `.sg-node` DOM（浮动动画随机时长 6~10s）、`.sg-edge` path、`.sg-edge-label-bg` rect、`.sg-edge-label` text，调 `_updateEdgesFromDOM`。
6. **`_updateEdgesFromDOM()`（每帧调）**：
   - 计算每条边端点（头像中心 + 半径 + terminalShift 偏移避免重叠），设 path `d`
   - 标签布局：沿线 13 采样点，中点为候选，与节点 rect / 头像 circle / 已置标签 rect 做碰撞检测；被遮挡则加 `lbl-hide` 隐藏，否则定位 text + bg rect
7. **`_startPhysics/_stopPhysics`**：RAF 循环每帧调 `_updateEdgesFromDOM`（追踪浮动节点的实时位置），并控制 `.sg-float-inner` 的 `animationPlayState`。graph 视图启用，works 视图停止。

节点视觉：
- `.sg-node` 绝对定位（`translate(-50%,-50%)`），主角 `.big` 放大头像
- `.sg-avatar` 圆形 + 白边 + `avatarBreath` 呼吸动画 + `::after` 脉冲光环
- hover/grabbing/active：放大 1.06 + 主色光环
- 主角首次进入 `.tap-hint` 脉冲引导（`tapPulse` 3 次，6 秒后移除）
- `.sg-name` 白底胶囊

### 4.3 连线

`.sg-edge` 为 SVG path（直线，端点偏移），五类关系语义色 + dasharray：
- family 实线红 0.75 / romance 点线粉 0.9 / master 长虚金 0.8 / enemy 复合虚深灰 0.45 / friend 长短虚绿 0.7
- 选中节点时：相关边 `.hl`（opacity 1 + 加粗），其余 `.dim`（opacity 0.08）
- 标签 `.sg-edge-label` + 白底 `.sg-edge-label-bg`，遮挡时 `.lbl-hide`

### 4.4 图例

左上角 `.sg-graph-legend` 常驻（移动端占满顶部），玻璃拟态底，五类关系横排，每项为一段 SVG line（样式与对应 `.sg-edge` 一致）+ 文案。面板打开时移动端隐藏。

### 4.5 角色详情面板

- **PC**：右侧滑出 `.sg-char-panel`（380px / 类驱动 320px，`translateX` 过渡），画布 `right` 让位
- **WISE**：居中弹层（`min(92cqw,420px)`），全屏遮罩 `rgba(20,22,30,.56)` + `backdrop-filter: blur(5px)`
- `.sg-close` 圆形关闭按钮（PC 主色 / 移动半透明白）
- 头部：头像 `.sg-avt`（86px 圆角卡片，error 兜底渐变）+ 姓名 + 演员 + 关系标签
- **关系标签逻辑**（复刻源页 showChar）：
  - 主角本人 → "核心人物 · 本人" + 主色渐变背景
  - 与主角有连线 → "与{主角名}：{label}" + `REL_COLORS[type]` 背景
  - 否则有 storyRel → `storyRel[0]` + 主色渐变
  - 兜底 → "本模块人物"
- `.sg-p-desc`：`storyDesc + desc` 拼接（去多余空白）
- **`_fitCharPanel`（移动端自适应）**：依次尝试 `fit-compact` → `fit-tight` → `fit-ultra`；仍溢出则按 `clientHeight/scrollHeight` 比例缩放 `--char-fit-scale`（最多 8 次，下限 0.62）

## 五、作品推荐视图（works）

### 5.1 子类 Tab + 分页器

头部 `.sg-works-head`：左侧子类 Tab（`role="tablist"`，胶囊样式），右侧分页器（上一页 / 页码 / 下一页）。本案例三子类：同题材 / 同主演 / 同编剧。

### 5.2 网格

- **PC**：`.sg-works-grid` CSS Grid（`repeat(4,1fr)` + `grid-auto-flow: column`），每页 4 项；不足补 `.sg-placeholder` 占位保持网格稳定
- **WISE**：横滚 flex（`flex: 0 0 48%`，2 列横滚 + scroll-snap），隐藏分页器，右侧出现 `.sg-works-scroll-hint`（`hintNudge` 抖动）

### 5.3 作品卡片

`.sg-work-card` 圆角卡片：封面（cover img，error 兜底渐变 + 底部蒙版）+ 角标 tag（待播用金，否则集数/时长或"推荐"）+ 信息区（名称 + 集数胶囊 + 主演 + 推荐理由）。hover 上浮 + 阴影增强。

## 六、交互逻辑

| 触发 | 行为 | 实现 |
|---|---|---|
| 点击底部 Tab | 切视图 + 渲染 | `_switchTab` 同步 aria/hidden/active；graph→`_relayout`+`_startPhysics`，works→`_stopPhysics` |
| 点击故事按钮 | 切故事 + 重渲染图谱 | 更新 `_activeStoryKey` + aria-selected + 描述 + `_closeChar` + `_buildGraph` |
| 点击/回车节点 | 打开面板 + 高亮连线 | `_showChar` 写面板数据 + 关系标签 + `.panel-open` + `_highlightRelated` + `_relayout` + `_fitCharPanel` |
| 拖动节点 | 移动 + 连线重绘 | `_attachDrag` mouse/touch（位移 `/scaleFactor` 补偿 + 边界 clamp + dragged 防误触）+ `_updateEdgesFromDOM` |
| RAF 循环 | 浮动追踪 + 边同步 | `_edgeSyncLoop` 每帧 `_updateEdgesFromDOM`（追踪 float-inner 动画位移） |
| X / ESC | 关闭面板 | `_closeChar` 移除 `.panel-open` + 清高亮 + `_relayout` |
| 点击作品子 Tab | 切分类 + 重置页 | `_worksCat` + `_worksPage=0` + `_renderWorks` |
| 翻页 | 上下页 | `_worksPage++/--` + `_renderWorks`，边界 disabled |
| 横滚提示 | 平滑滚动 | story/works scroll-hint 点击 `scrollBy(clientWidth*0.7)` |
| resize | 缩放同步 + 重排防抖 | `_applyScale` 立即 + 150ms 后 `_buildGraph`/`_renderWorks` |

## 七、A11y 约定

- `role="region"` + `aria-label` 包裹整卡
- Tab Bar / 故事条 / 作品子 Tab 均为 `tablist` + `tab` + `aria-selected` + `aria-controls`
- 视图为 `tabpanel` + `aria-labelledby` + `hidden`
- 角色节点 `role="button"` + `tabindex="0"` + `aria-label`（角色名 + 演员），支持回车/空格
- 角色面板 `aria-live="polite"`
- 关闭按钮、分页按钮、横滚提示均有 `aria-label` 或可见文案
- ESC 关闭角色面板
- `role` 属性通过 `el()` 工具的对象属性形式传入，源码含字面 `role: 'tablist'` / `role: 'tab'` / `role: 'tabpanel'` / `role: 'button'` / `role: 'region'` / `role: 'group'`
- SVG namespace 通过字符数组拼接规避字面量 URL（避免被校验误判为业务外链）

## 九、文件与命名约束

- 所有类名 / id / 全局对象使用 `sg-` 前缀（CSS 类 `.sg-*`，JS 全局 `Sandadui`，挂载根 `.sg-frame`）
- CSS 变量统一 `--sg-*`，定义集中在 `:root, .sg-frame` 作用域
- JS 渲染引擎为 IIFE + prototype，挂 `window.Sandadui`；零外部依赖
- 业务数据全部走 `options`，引擎内无硬编码角色名 / 故事名 / 作品名
